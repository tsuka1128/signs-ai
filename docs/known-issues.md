# 未実装事項・既知の制限

現在の Signs AI MVP における未実装機能と既知の制限事項をまとめます。

> **最終更新:** 2026-02-28

---

## 🔴 未実装（MVP 次フェーズで対応予定）

### 1. KPI 実績値の入力・保存

- **現状:** `/kpi` 画面は UI のみ存在し、`kpi_records` テーブルへのデータ保存が未実装
- **影響:** 月次 KPI の実績値が蓄積されない
- **対応予定:** `/kpi` 画面と `/api/kpi-records` API の実データ連携

### 2. アンケート（`/form`）の実データ連携

- **現状:** アンケートフォームの UI は存在するが、`survey_responses` / `survey_answers` テーブルとの連携が未確認
- **影響:** アンケート回答がデータベースに保存されない可能性がある

### 3. AI 診断・インサイトの自動生成

- **現状:** ダッシュボードのインサイト表示はダミーデータ固定
- **影響:** AI による実際の組織分析が行われない
- **対応予定:** Gemini / OpenAI API との連携による月次バッチ生成

### 4. アクション管理（`action_items`）

- **現状:** DB テーブルと RLS は定義済みだが、UI が未実装
- **影響:** アクションの作成・進捗管理ができない

### 5. resource_records（リソース・人件費データ）

- **現状:** テーブル定義のみ、入力 UI なし
- **影響:** 部署ごとの人件費・生産性スコアの算出ができない

---

## 🟡 既知の制限事項

### 1. ダッシュボードにダミーデータが残存

ログインしていない状態（または Supabase からデータ取得できない場合）、以下の箇所にハードコードされたダミーデータが表示されます：

| 画面 | 未連携のデータ | ファイル |
|---|---|---|
| `/`（マトリックス） | 部署・KPI（Supabase 取得失敗時のフォールバック） | `src/app/page.tsx` |
| `/`（インサイト・アクション） | AI診断・アクション提案 | `src/app/page.tsx` |
| `/`（体温スコア） | パルスサーベイスコア | `src/app/page.tsx` |

### 2. 未ログインでもダッシュボードが表示される

- `middleware.ts` で `/` はパブリックルートとして設定しているため、未ログインユーザーもアクセス可能
- その場合、Supabase からのデータ取得は失敗しダミーデータが表示される
- **注意:** 未ログインユーザーには機密データは一切表示されない（ダミーデータのみ）

### 3. トランザクションなし

- `/api/onboarding` の処理は複数の INSERT で構成されているが、途中でエラーが発生しても **ロールバックされない**
- 対応策：失敗した場合は Supabase ダッシュボードから手動でデータを削除し、再度オンボーディングを実行

### 4. 1社1ユーザー（マルチユーザー未対応）

- 現在の設計では 1 企業に複数ユーザーを紐づける想定だが、RLS のポリシーが `auth.uid()` に依存しているため、別ユーザーが同一企業にアクセスする仕組みが未整備
- 複数メンバーでの利用は現状サポートされない

### 5. middleware の警告

- Vercel ビルド時に `The 'middleware' file convention is deprecated. Please use 'proxy' instead.` の警告が表示される
- 動作への影響はなし
- 対応予定：Next.js 17 以降での migration 対応
