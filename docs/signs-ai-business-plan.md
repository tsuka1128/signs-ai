## エグゼクティブサマリー

Signs AIは、SaaS企業向けの「攻めの人的資本経営AI SaaS」である。既存のエンゲージメントサーベイ（ミキワメ、Wevox、ラフールサーベイ等）が「組織の健康診断」に留まるのに対し、Signs AIは**KPI達成（売上目標、採用計画、マーケティング指標）を起点に、AI経営参謀が攻めのアドバイスを行う**点で本質的に異なる。

「数字（KPI実績）× 人数（リソース）× 体温（現場の声）」を統合し、AIが階層別に140文字の断定的提言を生成する。経営者の孤独な意思決定を「データという後ろ盾」で支える。

## 1. Claudeの本音：このアイデアは売れるか

### 結論：条件付きでYES。ただし、楽観は禁物。

**売れる根拠（強み）**

- **空白地帯を突いている。** 既存サーベイツールは「守り（離職防止・メンタルケア）」に特化。KPI達成のために組織状態を「攻めの武器」として使うツールは、日本市場にほぼ存在しない。この「守り→攻め」の言い換えは、経営者のペインに刺さる。
- **月額5万円は絶妙。** Wevoxは最低月額9万円、ラフールサーベイは月額1.6万円〜（従業員数依存）。Signs AIの月額5万円・定額は「人数に関係なく使える」点で中小SaaS企業に導入しやすい。
- **AI診断の断定口調は差別化になる。** 既存ツールは「可視化して終わり」が大半。AIが「来月ここが腐る。今すぐ人を動かせ」と断言するのは、経営者が求めている体験。
- **1人開発×低コスト構成で損益分岐点が極めて低い。** 月5万円×10社＝月50万円で黒字化可能。

**売れない可能性（リスクと弱点）**

- **「サーベイ疲れ」問題。** 多くのSaaS企業は既にWevoxやSmartHRのサーベイ機能を使っている。「また別のアンケート？」という拒絶反応は必ず起きる。対策：既存ツールとの「共存」を明確に打ち出す（Signs AIはサーベイツールではなく「AI経営参謀」である、というポジショニング）。
- **KPI入力の運用負荷。** 毎月CSVやスプレッドシートにKPIを手入力する運用は、忙しい管理者にとって地味に重い。最初の3ヶ月で「面倒だから使わなくなった」となるリスクが高い。対策：入力画面を極限まで簡素化し、前月値のコピー機能を入れる。
- **1人運営のCS限界。** SaaS企業の経営層は「ツールの使い方」より「診断結果の解釈」にCSを求める。1人でこの手厚さを維持できるのは10〜15社が限界。
- **AI診断の精度問題。** セマンティックレイヤーの記述が浅いと、AIの提言が的外れになる。最初の数ヶ月は「AIが的外れなことを言う」体験が発生し、解約につながるリスクがある。

### 正直な勝率評価

| 指標 | 評価 |
|---|---|
| 市場ニーズの有無 | ★★★★☆（ペインは確実にある） |
| 競合との差別化 | ★★★★☆（明確に異なるポジション） |
| 初期顧客獲得の難易度 | ★★★☆☆（SaaS経営者ネットワークが鍵） |
| 継続率（チャーン耐性） | ★★☆☆☆（運用負荷とAI精度が課題） |
| 1人運営の持続性 | ★★★☆☆（15社までは可能、それ以上は人が必要） |

## 2. 市場環境

### 2-1. マクロ市場

国内SaaS市場は年平均成長率約11%で拡大を続け、2029年度には3.4兆円規模に達する見通し。「SaaS is Dead」の議論はあるが、AIとの統合で市場は変革期に入っており、AI×SaaSの新カテゴリは投資家・企業双方から注目されている。

### 2-2. ターゲット市場（TAM → SAM → SOM）

| 層 | 定義 | 規模 |
|---|---|---|
| TAM | 日本国内の従業員20名以上の企業（SaaS・IT・コンサル・VC含む） | 推定5,000〜8,000社 |
| SAM | うちKPI管理・組織改善に課題を持つ企業 + 顧客組織を診断したいコンサル・VC | 推定1,500〜2,500社 |
| SOM | 初年度に現実的にリーチ・獲得可能な企業 | 15〜25テナント |

**ターゲット顧客の決裁者像：**

| セグメント | 決裁者 | 導入動機 |
|---|---|---|
| SaaS企業（50〜300名） | CEO / 経営企画 / HRBP | 自社のKPI達成と組織体温の統合管理 |
| スタートアップ（20〜50名） | CEO / 人事責任者 | 急成長期の組織崩壊を予防しながら攻める |
| コンサルティングファーム | パートナー / マネージャー | クライアント企業の組織実態を定量把握。支援提案の精度向上 |
| VC / CVC | 投資担当 / バリューアップ担当 | 投資先ポートフォリオの組織健全性モニタリング |

### 2-3. 競合分析

| サービス | ポジション | 価格帯 | Signs AIとの違い |
|---|---|---|---|
| Wevox | エンゲージメント可視化 | 月額9万円〜（最低150名分） | 「守り」の健康診断。KPIとの統合なし |
| ミキワメ | ウェルビーイングサーベイ | 要問合せ | 個人の性格×心理状態。組織KPIとの接続なし |
| ラフールサーベイ | 組織診断・ストレスチェック | 月額1.6万円〜 | 厚労省準拠の守り寄り。AI提言機能が弱い |
| モチベーションクラウド | エンゲージメント診断+コンサル | 高価格帯 | コンサル込みの重厚モデル。セルフサービスでない |
| **Signs AI** | **KPI×体温のAI経営参謀** | **月額5万円（定額）** | **攻めのKPI達成を起点。AIが断定的に提言** |

### Signs AIの独自ポジション

```
          守り（離職防止・メンタルケア）
              ↑
    ミキワメ ● ── ● ラフール
              |
  Wevox ●    |
              |
  ──────────────┼──────────────→ 攻め（KPI達成・組織変革）
              |
              |        ★ Signs AI
              |
    モチクラ ●

          重厚（コンサル依存）
```

**Signs AIは「攻め × セルフサービス」の空白地帯を狙う。**

## 3. ビジネスモデル

### 3-1. 料金プラン

| プラン | 月額（税別） | 対象 | 主な機能 |
|---|---|---|---|
| **Team** | 2万円 | 20名以下の単一チーム・部署 | KPI 5個まで / 部署1つ / AI診断（現場向けのみ） |
| **Organization** | 5万円 | 組織全体（部署横断） | KPI 10個 / 部署無制限 / 4階層AI診断 / セマンティックレイヤー |
| **Enterprise** | 10万円 | 複数事業部・経営企画が主導 | Organization全機能 + 部署間360度分析 / 診断エクスポート / 優先サポート |
| **Partner（特別プラン）** | 15万円〜（応相談） | コンサル・VC（複数クライアント管理） | 複数テナント一括管理 / クライアント横断ベンチマーク / オンボーディング外販権 |

**Partnerプランの狙い：** コンサルティングファームやVCは、投資先・支援先企業の組織実態を定量的に把握したいニーズがある。複数テナントを横断で閲覧できるダッシュボードを提供し、「クライアントの体温を一覧で見る」体験を作る。さらに、コンサル自身がSigns AIのオンボーディングをクライアントに外販できるライセンスを付与し、販売チャネルとして機能させる。

### 3-2. 価格設計の思想

- **Teamプラン（2万円）** はエントリー。小さなチームで「AI診断の体験」をさせ、組織全体への導入（Organization）へアップセルする階段。
- **Organization（5万円）** がメインの収益源。経営者・人事・HRBP・経営企画が導入決裁者。
- **Enterprise（10万円）** は複数事業部を持つ企業向け。部署間360度分析が差別化の鍵。
- **Partner（15万円〜）** はコンサル/VCの「クライアント管理ツール」として売る。彼ら自身の付加価値サービスになるため、価格感度が低い。
- 1人運営で月間固定費15〜20万円（後述）なら、**Organizationプラン4社で損益分岐点を突破**。

## 6. 損益分岐点分析

```
月間固定費: 2万円（インフラのみ）
月間固定費: 20万円（生活費込み）

損益分岐点（インフラのみ）: 2万 ÷ 5万 = 0.4社 → 実質1社で黒字
損益分岐点（生活費込み）:  20万 ÷ 5万 = 4社 → 4社で生活可能
```

**4社獲得が最初のマイルストーン。** これを達成できなければ、プロダクトかセールスのどちらか（または両方）に根本的な問題がある。

## 8. 競争優位性（なぜ大手に真似されないか）

### 正直に言えば、機能だけなら真似できる。

Wevoxが「KPIとの統合」を追加すれば、技術的には似たものは作れる。しかし：

1. **既存サーベイツールはポジション変更できない。** Wevoxやミキワメは「従業員の心理的安全性を守る」ブランドで成長した。急に「攻めのKPI達成ツール」を名乗れば、既存顧客が混乱する。
2. **セマンティックレイヤーが真の堀になる。** 顧客の経営方針・フェーズ・KPIの解釈が蓄積されるほど、AIの診断精度が上がり、スイッチングコストが高くなる。これは時間と信頼の蓄積であり、後発には真似しにくい。
3. **「断定する」AI。** 大企業のSaaSは訴訟リスクを恐れて「示唆」に留まる。スタートアップだからこそ「来月ここが腐る」と断言できる。

## 9. リスクと対策

| リスク | 発生確率 | 影響度 | 対策 |
|---|---|---|---|
| AI診断が的外れ | 高 | 高 | 後述「AI診断の精度保証設計」を参照 |
| KPI入力が面倒で使われなくなる | 高 | 高 | 前月値コピー機能、入力リマインド通知、V2でSFA/CRM自動連携 |
| アンケート回答率が低い | 中 | 高 | 後述「体温のある語りかけ設計」を参照 |
| 競合の追随 | 低 | 中 | セマンティックレイヤーの蓄積でスイッチングコストを上げる |
| 1人運営の限界 | 中 | 中 | Phase 3でオンボーディングを外部委託、Phase 4で完全委譲 |
| Google Sheets APIの制限 | 低 | 中 | V2でFirestore/Supabaseへのマイグレーションパスを事前設計 |

### 9-A. AI診断の精度保証設計（的外れにならない仕組み）

AI診断が「的外れ」になる主な原因と、それぞれの構造的対策を以下に定義する。

**原因1：セマンティックレイヤーが空 or 浅い**
→ AIが企業の文脈を知らないまま汎用的な提言をしてしまう。

| 対策 | 詳細 |
|---|---|
| 段階的テンプレート | 空欄のまま保存させない。最低限の5問（フェーズ/最重要KPI/体温の解釈/地雷ワード/今月の最優先事項）に答えるだけで初期セマンティックレイヤーをAIが自動生成 |
| オンボーディング必須化 | 初回設定時に30分の対話セッション（Phase 3以降はPartnerコンサルが代行）。ここでセマンティックレイヤーの骨子を一緒に作る |
| 月次リフレッシュ | 集計実行時に「先月の方針から変わったことはありますか？」の1問を管理者に表示。変更があればセマンティックレイヤーを部分更新 |

**原因2：データ不足（KPI未入力 / アンケート回答率低い）**
→ AIが限られた情報から無理に推論し、飛躍した提言をする。

| 対策 | 詳細 |
|---|---|
| 「自信度」表示 | AI診断に「この診断の確信度: ★★★☆☆」を付与。入力データが少ない場合はAI自身が「データ不足のため断定を控えます」と明示する |
| 最低データ閾値 | KPI入力が3項目未満 or アンケート回答率30%未満の場合、診断生成を保留し「データが足りません」と表示。的外れな診断を出すより「出さない」方がマシ |
| 欠損データの明示 | 「営業部の商談数が未入力のため、営業部の診断精度が低下しています」とAIが具体的に不足データを名指しする |

**原因3：AIの「もっともらしい嘘」（ハルシネーション）**
→ 実際のデータにない因果関係を捏造する。

| 対策 | 詳細 |
|---|---|
| 根拠の強制引用 | プロンプトで「診断には必ずt_kpi_logsまたはt_survey_responsesの具体的な数値を1つ以上含めること」と制約。数字の裏付けがない提言を構造的に排除 |
| ファクトチェック層 | AI診断生成後、別のClaudeコールで「この診断は入力データと矛盾していないか？」を検証するダブルチェック機構を実装 |
| 管理者フィードバック | 各診断に「👍 的確 / 👎 的外れ」ボタンを設置。👎が押された場合、その診断とセマンティックレイヤーをログに記録し、プロンプト改善に活用 |

**原因4：階層別トーンのミスマッチ**
→ 経営層向けの冷徹なトーンが現場に表示される、等。

| 対策 | 詳細 |
|---|---|
| 出力前のロール検証 | 生成された診断テキストに「対象ロール外の情報（例：現場向け診断にリソース配分の話）」が含まれていないかをルールベースでフィルタリング |

### 9-B. 「体温のある語りかけ」設計（アンケートUX）

アンケートは「業務」ではなく「対話」であるべき。回答者が「自分の声が届く実感」を持てるよう、以下のUX設計を適用する。

**設計原則：「30秒の深呼吸」**
アンケートを開いた瞬間から回答完了まで、一貫して「あなたの声を聴きたい」という温度を伝える。

**① 冒頭の語りかけ（アンケート開始画面）**

> 今月もお疲れさまです。
> これから11の問いをお届けします。
> 正解はありません。今のあなたが感じていることを、そのまま。
> 所要時間は約2分。深呼吸ひとつ分の時間をください。

**② 設問ごとの「ひと言添え」**

各設問の下に、小さなグレーテキストで「語りかけ」を添える。

| 設問 | 語りかけ |
|---|---|
| Q1. 今の仕事にワクワクしていますか？ | *月曜の朝、布団の中で思い浮かべてみてください。* |
| Q2. 何かが決まるのを「待つ時間」は少なかったですか？ | *待っている間、あなたの熱量は少しずつ冷めていきます。* |
| Q3. 必要な情報が自分まで届いていましたか？ | *知らなかったことで、損をしていませんでしたか。* |
| Q4. 「調整」や「根回し」に時間を取られすぎていませんか？ | *本来、何に使いたかった時間ですか。* |
| Q5. 言いづらいことを飲み込まずに伝えられましたか？ | *飲み込んだ言葉は、どこかで体温を下げます。* |
| Q6. 「何を成すべきか」に迷わず集中できましたか？ | *迷いは、あなたのせいではないかもしれません。* |
| Q7. 上司や仲間から反応（賞賛や指摘）がありましたか？ | *無反応は、じわじわと人を蝕みます。* |
| Q8. 業務量は、質を維持できる範囲でしたか？ | *「頑張ればできる」は、長くは続きません。* |
| Q9. 「顧客のプラスになること」に時間を使えましたか？ | *社内都合に時間を奪われた日、悔しくなかったですか。* |
| Q10. 新しい工夫や挑戦ができましたか？ | *同じことの繰り返しは、安全に見えて危険です。* |
| Q11. KPI達成に向けて、準備周到に活動できていますか？ | *道筋が見えているだけで、体温は上がります。* |

**③ ボトルネック選択時の語りかけ**

> 「数字を止めている原因」に心当たりがあれば、教えてください。
> あなたの直感は、データよりも早く真実を捉えていることがあります。

**④ 自由記述前の語りかけ**

> ここから先は、あなたの言葉で。
> AIが責任を持って、あなたの本音を「建設的な提言」に翻訳し、届けます。
> 名前は絶対に出ません。安心して、書いてください。

**⑤ 回答完了画面**

> お疲れさまです。あなたの声、確かに受け取りました。
> この2分間が、組織の体温を少し変えるきっかけになるかもしれません。
> また来月、ここでお会いしましょう。

```
[コンサル/VC]（Partnerプラン契約者）
  │
```

## 付録：Claudeからの辛口メモ

> **最大のリスクは「作って満足すること」です。**
>
> 技術的にはAntigravityで美しいMVPを作ることは十分可能です。しかし、SaaSの墓場には「プロダクトは美しいが顧客がいない」サービスが山ほどあります。
>
> Phase 0（手動検証）を絶対にスキップしないでください。Claude APIのプロンプトを磨いて、Googleスプレッドシートに手入力したデータから「模擬140文字診断」を3社分作り、「これに金を払うか」を聞く。これが最も安く、最も確実な検証です。
>
> もう1つ。月額5万円は「安い」と書きましたが、SaaS企業の経営者は「月5万円のツールを10個使っている」のが現実です。彼らは新しいツールを入れるとき「どれか1つを辞めて入れ替える」判断をします。Signs AIが「何と入れ替えるに値するか」を言語化できないと、トライアルで終わります。
>
> 売れる確率を上げる最大のレバーは、**セマンティックレイヤーの初期設定を創業者自身がオンボーディングで一緒にやること**です。ここでの会話が顧客との信頼関係を作り、AI診断の精度を上げ、チャーンを防ぎます。スケールしないことを最初にやる。それが1人SaaSの正攻法です。
